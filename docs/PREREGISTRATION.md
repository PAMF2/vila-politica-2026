## 1. Summary

This document is a pre-registration of probabilistic forecasts for the
Brazilian 2026 general elections (presidential, gubernatorial, senatorial),
generated by the Vila INTEIA `political_cohort` model with MRP state
baseline (v1.2). The model achieves **97.21% accuracy** under year-fold
cross-validation (T <= 30 days, n=394) on six historical BR election cycles
(2010, 2016, 2018, 2020, 2022, 2024).

The purpose of this pre-registration is to **lock predictions, methodology,
and decision rules before any 2026 outcomes are observed**. All artifacts
are content-addressed (SHA-256). Post-election, we will publish a
quantitative outcome evaluation regardless of result (no file-drawer).

## 2. Hypotheses

### H1 - Aggregate accuracy

> The Vila MRP model's average accuracy on actual 2026 outcomes
> (presidente + governador-by-UF + senador), restricted to events with
> T <= 30 days from forecast freeze, is **>= 95%**.

- Operationalization: `acc = (#correct top-pick) / (#offices with TSE result)`.
- Top-pick rule: candidate with `max p_winner` per office (see §5).
- Senate seat counted only if a unique top-pick exists per UF.

### H2 - SP gubernatorial (Tarcísio reelection)

> Conditional on `p_winner(Tarcísio, SP) >= 0.60` in the frozen
> snapshot, Tarcísio de Freitas wins the 2026 SP gubernatorial election.

Frozen snapshot: `p_winner(sp_2026_tarcisio) = 0.6509` (>= 0.60).
Hypothesis is fixed; outcome is a single Bernoulli observation.

### H3 - Lula 1st-round lead persistence

> If Lula's 1st-round national lead over the runner-up is **>= 5 pp**
> (per the latest aggregated polls registered in the frozen snapshot),
> Lula wins the 2nd round.

This tests the persistence of 1st-round leads >= 5pp into the runoff,
which historical BR data (2002, 2006, 2010, 2014, 2022) supports at
~85% empirical rate.

### H4 - Right-wing unification implies right-wing runoff win

> If `p_winner(Lula) < 0.50` AND the sum of probability mass on
> right-wing presidential candidates (Tarcísio + Ratinho + Zema) in the
> frozen snapshot is **>= 0.50**, then a right-wing candidate wins the 2nd
> round.

Frozen snapshot: `p_winner(Lula) = 0.2479`, sum right-wing =
`0.2165 + 0.1927 + 0.1912 = 0.6004` (>= 0.50). Hypothesis is fixed.

## 4. Methodology (frozen)

The forecasting pipeline is identical to `scripts/predict_2026.py` at the
SHA above. Conceptual flow:

1. **Cohort empirical-Bayes prior** `p_cohort` from
   `engine/political_cohort.py::predict_political`. Cohort key:
   `(cargo, days_bin, lead_bin, incumbente, regime)`. Fallback chain:
   `full -> (cargo, days_bin) -> (cargo,) -> _global`. Stein shrinkage
   `lambda = 0.05` toward global rate.
2. **Linzer (2013) state-space win-probability** `p_linzer = Phi(lead_pp /
   (sigma_intercept + sigma_slope * days))` with `sigma_intercept = 4.0 pp`,
   `sigma_slope = 0.05 pp/day`.
3. **MRP state baseline** `p_state` from Laplace-smoothed (UF, regime)
   posteriors over 2010-2024 BR cycles.
4. **Ensemble blend**:
   ```
   p_winner = (1 - w_linzer) * p_cohort + w_linzer * p_linzer
   ```
   with `w_linzer = 0.5`, `w_state_mrp = 0.36` applied additively for the
   gubernatorial tier (`tier == "global"`).
5. **Operational filter**: only events with `horizon_days <= 30` enter the
   primary H1 evaluation.
6. **Selective tau gating** (optional, reported separately): predictions
   with `|p_winner - 0.5| >= tau` are flagged "high-confidence". Validated
   tau values: `0.15` (96.1% acc, 92% coverage), `0.40` (100% acc, 11%
   coverage).

House effects (per-pollster bias) were tested and disabled by default
(0.9416 -> 0.9391 marginal degradation, see
`data/political_best_config.json::notes`).

## 5. Decision rule

For each office (presidente, governador-by-UF, senador-by-UF), the
top-pick is the candidate with `max p_winner` in the frozen snapshot.

Tie-breaking (probability difference < 1e-3):
1. Higher `p_cohort`.
2. Higher `p_linzer`.
3. Lexicographic by `registry_id`.

A prediction is "scored correct" iff the top-pick wins the corresponding
TSE-certified contest (1st round for governadores/senadores; 2nd round
for presidente if applicable, otherwise 1st round).

## 7. Outcome evaluation protocol

Post Oct 4, 2026 (1st round) and Oct 25, 2026 (2nd round if applicable):

1. Fetch official TSE results from `https://resultados.tse.jus.br`.
2. Score each frozen prediction against TSE outcome (correct / incorrect).
3. Compute aggregate accuracy, Brier, log-loss, calibration curve, and
   selective-coverage curves.
4. Test H1-H4 against pre-registered thresholds.
5. Publish `docs/POSTMORTEM_2026.md` with full results table + scripts to
   reproduce, signed git commit on tag `v1.2-postmortem`.

## 8. Pre-commitments

- We will **not** modify the model, configuration, or prediction
  snapshot between this pre-registration date (2026-05-07) and the
  election (2026-10-04).
- We will **publish the post-mortem regardless of outcome** - including
  cases where the model performs below H1's 95% threshold or any of
  H2-H4 fail. No file-drawer.
- **Code freeze**: a signed git tag `v1.2-prereg` is created at git HEAD
  `7d2403b7dd5756f95b378331e43405cae60e62da`. Any subsequent change to
  the model or snapshot is tracked as a separate tag and disclosed in
  the post-mortem.
- Selective gating thresholds (tau) for H1 will use the values committed
  in `data/political_best_config.json` (`tau_0_15`, `tau_0_40`). Choosing
  a different tau post-hoc is disallowed.
